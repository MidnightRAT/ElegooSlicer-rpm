# Fix GitHub Action Workflow - Build RPM Step Failure

## Problem

Workflow fails at step 9 "Build RPM and SRPM". Root causes identified:

1. **Tag format mismatch**: GitHub tag is `v1.5.2.2` but spec expects version `1.5.2.2`
2. **Source tarball directory name**: GitHub tarballs extract to `ELEGOO-3D-ElegooSlicer-<sha>`, not `ElegooSlicer-{version}`
3. **Complex build from source**: Requires many dependencies and can fail for various reasons

## Solution

Use the pre-built AppImage from GitHub releases instead of building from source:
- AppImage URL: `https://github.com/ELEGOO-3D/ElegooSlicer/releases/download/v1.5.2.2/ElegooSlicer_Linux_Ubuntu2404_V1.5.2.2.AppImage`

## Files to Modify

### 1. `.github/workflows/build-rpm.yml`
- Remove all build dependencies (not needed for AppImage)
- Download AppImage from release instead of source tarball
- Extract AppImage and package into RPM
- Much simpler and faster workflow

### 2. `elegoo-slicer.spec`
- Rewrite to package pre-built AppImage contents
- Remove BuildRequires (no compilation needed)
- Keep Requires for runtime dependencies
- Install to `/opt/ElegooSlicer`

## New Workflow Steps

1. Checkout repository
2. Install minimal deps: `rpm-build`, `rpmdevtools`, `wget`, `file`, ` fuse-libs` (for AppImage extraction)
3. Get latest release version (handle `v` prefix)
4. Download AppImage
5. Extract AppImage: `./ElegooSlicer*.AppImage --appimage-extract`
6. Prepare RPM build tree
7. Create spec file with correct version
8. Build RPM and SRPM
9. Upload artifacts

## Verification

After fixing, push to GitHub and verify:
1. Workflow completes successfully
2. RPM and SRPM artifacts are uploaded
3. RPM can be installed and ElegooSlicer runs
